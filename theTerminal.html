The Terminal

We can interact with our computer through the terminal instead of the GUI.

Why it matters:

Speed, it can be much faster than the GUI.(NO CLICKING,DRAGGING,ETC.)

Access, we get access to areas we normally can't interact with or are hidden from us.

Tools, many of them are installed through the terminal, not much choice.

Terminology

The terminal is a text-based interface to our computer. (The actual application.)

It was originally a physical object, now a software.

A shell is the program running on the terminal. (An ATM is the terminal, and the software each bank uses is each shell.)

Bash is a popular shell, the default on Mac in the past. We can change between shells on our terminals.

Commands

ls , list

We are in the default directory when we open a new terminal, it is our Home (~ is the shortcut.).

With ls, we can list all the directories and/or files in our current location.

pwd (Print Working directory)

It tells us exactly where we are in our files.

cd (change directory)

We use this command to move into or back out to a folder/file.

We need to specify where we are trying to go with a path.

Tab will auto-complete our path if it matches anything.

cd.. will take us backwards one step.

Absolute vs Relative paths

With a relative path, we can only access a specified path moving into or forward if the file is available relative to where we currently are.

  • If dogs has a file inside it called germanShepard, we can only access it if we are ALREADY inside the dogs folder.

With an absolute path, we can go anywhere else, also moving forward.

If we are outside dogs, we can specify an absolute path.

  • If we are in animals, we can access germanShepard with:
  • cd /dogs/germanShepard

mkdir (make directory)

It will create a new folder wherever we are in the file path.

The name of the folder is followed by the command, just separated by a space.

If there is a folder with the same name as the one we are trying to create, we will be told so.

It can be combined with .. and ../.. , etc.

man (Manual)

It provides information about the different commands.

Flags

They are options and parameters to specify how we want the command to run, what info to display, the order,etc.

They start with a dash.

    Examples:
  • ls -a shows us all entries, including hidden filles which begin with a dot/period.
  • -l gives more details.
  • -la does both things (THEY CAN BE COMBINED!)
  • mkdir -v tells us what was created, instead of doing it silently like normal.

We can exit by typing q.

touch

It helps us create a file, in contrast to mkdir, which creates a folder or directory.

It's primary use is making a new file, but it is originally to change file access or modification times (Last modified update).

If we use touch on an existing file it will do just that.

We can specify the type of file, for example:

  • touch newDoc.html
  • touch newDoc.html
  • touch newDoc.html
  • (OR --> touch newDoc.html newDoc.html newDoc.html)
  • man touch (opens the manual for the touch command :D)

Tips

We can clear the terminal with clear or Ctrl L on Linux.

cd .. takes us one directory back.

cd ../.. takes us two directories back.

cd ../../something takes us to a directory 2 levels outside, but inside that particular level.

cd ~ takes us home from wherever we are.
