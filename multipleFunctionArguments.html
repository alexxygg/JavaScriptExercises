Multiple arguments

INstead of just one argument, we can create our functions to expect multiple arguments when we define it.

What's important is the order in which we define the arguments, since that is the order in which they will be worked with.

Here we have an example.

We have created a function that takes 4 arguments. We have called the function 5 times, the last time with the arguments not in order.

Here the result is not that bad, sometimes we have arguments of different types, like a string followed by an array and then a boolean, where things can get much worse.

Examples

Repeats a string in separate strings x number of times.

Repeats a string in a same string x number of times.

Concatinates strings from user input to create a single string.

Calculator to sum up to 4 numbers, to get a number to the 3rd power and to the 4th power.(7*7*7 / 7*7*7*7)

Tells you the amount of minutes in x number of days.

Checks if the 2 inputs are 1s to log a message, otherwise logs another message.
