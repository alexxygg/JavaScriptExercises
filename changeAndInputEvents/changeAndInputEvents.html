Change & Input Events

change Event

This event will check and perform an action every time the user makes a change to the input, as long as they click away from the input field.

If the user clicks the input and clicks away without making any changes, the event WILL NOT be triggered.

(It works when user leaves the input and there are new changes, not when something changes or happens on the user's screen.)

Other keys like arrows and shift will not trigger the event either.

We could use key or value for a keydown event, but it would register any key, not just characters accepted on the current input.

input Event

Here we will run the code we want every time there is aan actual change to the input, be it a single character, pasting something, text to speech, etc.

It triggers the event whenever anything inside the input changes.

(clicking away without changes, arrow keys, shift key, etc. will not trigger event, just like with change event listeners.)

It actually doesn't even check for leaving the input field/clicking away, only for changes to the field!

We can have it work as a live preview on the screen.
