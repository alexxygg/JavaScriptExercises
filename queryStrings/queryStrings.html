Query Strings

We may see endpoints with classes on API documentation that are used to specify the id that we are requesting, which is to be replaced by us.

Query Strings show on our address bars and have the following syntax:

?sort=desc&color=blue

They are key value pairs separated by ampersand characters.

This address tells us that the value of sort is desc AND also that the value of color is blue.

We can append anything to our strings when making a request, anything that we add to it that the API database doesn't recognize will simply be ignored.

Example:

https://swapi.dev/api/starships/10/?color=blue&details=true&random=yes

Only https://swapi.dev/api/starships/10/ will be used.

It won't look for the rest, it doesn't care.

Some APIs will only accept requests made with their specific query selectors, and will otherwise not work.

We can instead enter query parameters if we don't want to enter the url manually with all the specifications. We can also toggle the parameters this way.

HTTP Headers

They allow us to pass key value pairs, small add-ons for our request.

It is information returned by the server regarding the request, some APIs require headers.

The documentation lets us know of the specifications we must fill for the key value pairs, if it is needed.
