AJAX

Asynchronous JavaScript And XML

When we are loading new posts on a social media, we don't need to go to another webpage, we simply get an infinite scroll type of system.

There's no need to click or move from the site.

Autocompletion and suggestions work the same way, the matches are provided by the pageÂ´s server, and requested by the page itself based on the user's input.

We can fetch all sort of data behind the scenes, all while on the same page. Just the bits we need, nothing more, nothing less.

We don't get HTML, CSS or JavaScript, just the data we ask for.

API's

Web API's are web based.

It is like a portal to another database or software, a way for them to communicate between each other.

An endpoint is the place where we send the request to, it not a website but JSON. What we are returned is not for a user, it is a way to return and have our code understand the values we ask for.

Many have a cost, others are free.

  • Weather data (current,forecasts,etc.)
  • Stock market prices
  • Twitter (Search tweets, post tweets bot, auto-responses to DMs/Post mentions)
  • Facebook, Instagram (Similar to above)
  • Twilio (Send text messages, whatsapp chat auto-responding, appointment reminders,etc.)

So sometimes it's not just to fetch data, but to create bots and display information.

These API requests and responses work through HTTP requests.

The format they use is pure data, no events, markup, etc.

The most popular was XML, but they aren't required anymore. It is far less common now.

Which is why we may see AJAJ instead, which replaces XML with JSON.

JSON

Java Script Object Notation

It is a format to send information, between browsers, API's, in a predictable and simple way.

Syntax

It works like JS objects, but with quotes between the key name.

No undefined, since it would convert it to a string as it doesn't recognize it as a type. It becomes invalid JSON if we check with a JSON validator.

There is a built-in method that converts JSON to JS objects, since what we are returned are strings.

We can then save the object to a variable and access its keys and values:

JSON.parse(string{})

We can also do the opposite:

JSON.stringify(object)

Postman

HTTP Verbs

These are the different options when creating a request, like get for example.

Some APIs only support a certain type of request, o our retrieve of information won't work if we don't follow the APIs documentation.

They do different things and they depend on the APIs structure.

HTTP Status Codes

These are the codes that return from an HTTP response, each has a meaning.

200 is the most common, it means the request succeeded. The ones that start with 2 mean success.

The ones that start with 3 mean something went wrong with the redirection of the request.

The ones that start with 4 mean something went wrong on the client side, or by the user, like 404 Not found.

The ones that start with 5 mean something went wrong on the server side.
