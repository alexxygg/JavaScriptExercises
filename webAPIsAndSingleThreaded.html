Web API'S and JS being Single Threaded

JS only has one thread, meaning it can not multi-task, only switch between tasks.

It only works in one thing at the time.

When we use a setTimeout function, the browser helps us.

Browsers come with Web API's that can handle certain tasks in the background

The JS call stack recognizes these API functions and passes them off to the browser.

It's as if JS asked the browser for a reminder to do a certain task after certain time, while it does other things to keep productivity at peak.
